# Оцифровка истории обработок — как подготовить данные

Цель: собрать историю полевых обработок (что, когда, чем, против чего, с каким
результатом) за 2–3 сезона, чтобы цифровой агроном опирался на реальную историю
поля. Это **наши собственные данные** — выгрузка из CropWise/1С или ручное
заполнение; никаких сторонних материалов.

## Два способа дать данные
1. **Заполнить шаблон** `catalog/treatment_history_template.csv` (Excel → «Сохранить как» → CSV).
2. **Выгрузить из CropWise / 1С** в Excel/CSV и переименовать столбцы под шапку ниже
   (либо прислать как есть — мы подгоним столбцы под импорт).

## Столбцы (шапка файла; разделитель — точка с запятой `;`)
| Столбец | Обязателен | Пример | Примечание |
|---|---|---|---|
| `поле` | да | Поле 121/140 | как в боте; если поле вне пилота — пишите как есть |
| `дата` | да | 15.05.2026 | формат ДД.ММ.ГГГГ |
| `культура` | желательно | Соя | что росло на поле |
| `операция` | желательно | опрыскивание | опрыскивание / внесение / обработка семян … |
| `препарат` | да | Селект | торговое название (свяжется с госкаталогом) |
| `действующее вещество` | нет | Клетодим | если знаете |
| `объект` | желательно | пырей ползучий | против чего (сорняк/болезнь/вредитель) |
| `норма` | да | 1,5 л/га | как в документах (запятая в норме — ок) |
| `площадь_га` | нет | 121 | обработанная площадь |
| `фенофаза` | нет | кущение | фаза культуры |
| `условия` | нет | пасмурно | погода/условия |
| `стоимость` | нет | | если ведёте |
| `результат` | нет | хороший эффект | наблюдаемая эффективность |
| `исполнитель` | нет | Алмас | кто проводил |
| `примечание` | нет | | любое уточнение |

Заполнять можно не все столбцы — чем больше, тем лучше, но **поле + дата +
препарат + норма** уже дают пользу. Один файл = много строк (по одной на каждую
обработку каждого поля).

## Как прислать файл
- **Taildrop** (между Вашими устройствами): `tailscale file cp history.csv …`
- **или** передайте файл Алексею — он положит его на сервер, и мы запустим импорт.

Импорт запускается командой `python -m catalog.ingest_treatments history.csv`
(добавляет записи; `--replace` — если нужно перезалить с нуля). Несовпавшие
названия полей сохраняются как текст и при желании поправляются позже.
