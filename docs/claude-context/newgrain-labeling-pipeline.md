The labeling pipeline (spec Часть II §2.x) turns collected photos into a CV training dataset. The **разметчик works in CVAT** (a web app), NOT the Telegram bot. Decided to use **CVAT Cloud (app.cvat.ai)** — managed, lowest-ops — to start, over self-hosting.

**CVAT access facts (where the tasks live):** host `app.cvat.ai`, **organization `NewGrain`**, project `weeds-diseases-stress` (id 405682), token/owner account **akhaliullin@me.com**. Tasks are named `batch-YYYYMMDD`. **Gotcha (hit 12 Jun 2026 — "can't see new photos in CVAT"):** CVAT opens in the **Personal workspace** by default, but the bot creates tasks in the **NewGrain org** — so the annotator must switch the workspace (top-right switcher) from "Personal workspace" → **NewGrain** to see them (or use direct `/tasks/<id>` links). A different login only sees them if it's a member of the NewGrain org. Nothing was broken — the export had worked (11 frames in 2 tasks, status `annotation`).

**CVAT Cloud free-tier task cap + jam incident (June 2026).** CVAT Cloud free tier caps the **org at 3 concurrent tasks** (POST /api/tasks → 403 "organization has reached the maximum number of tasks" once full). Another session added a nightly cron (`labeling/pipeline.sh`, 03:30, crontab) running `export` then `import --auto`. It created one task/day, filled all 3 slots by ~June 4, then every export 403'd → June 8-9 photos stranded at ready_for_labeling. **Fixed 10 June 2026** by rewriting `import --auto` (`labeling/import.py`): (1) **recycle** — a task marked 'completed' is (re)imported then DELETED to free the slot; (2) **rescue** — any task with annotations gets its labels pulled even if NOT marked 'Completed' (kept, not deleted) — fixes a stranding bug where the June-2 batch had 14 boxes that auto-import skipped because the annotator never clicked Completed. Import is idempotent (DELETE+INSERT labels). **Self-host evaluated and declined for now** — Yandex VM (~₽5-9k/mo + ops) is *more* expensive than CVAT Cloud Solo paid (~$23-33/mo, unlimited tasks); and CVAT only ever receives field images + manifest (no personal data → not a 152-ФЗ issue), so the RU-residency benefit of self-host is soft. Plan: stay free-tier + recycling; revisit self-host (on Yandex, not Google) at scale / when RU residency becomes a hard requirement. **Operational note for the annotator: press "Finish the job" in CVAT when done → imported + slot recycled on the next 03:30 run.** (Gotcha fixed 10 Jun 2026: "Finish the job" sets the JOB *state*='completed' but the TASK *status* stays 'annotation' in a single-stage workflow — recycle detection must check job state via `/api/jobs`, NOT task.status. Also: completed-path re-imports only when pending>0, so a manual DB de-dup/edit isn't clobbered.) Cron flow & all-statuses: ready_for_labeling → in_labeling → labeled.

**3-stage plan** (cheapest-useful-first; pre-annotation is an accelerator, not a prerequisite):
- **Stage 1 — CVAT up + annotator labeling.** ✅ Schema built: `labeling/cvat_labels.json` = 31 classes from taxonomy §2.3 (15 weeds + 10 diseases as bboxes, 6 stresses as tags); `labeling/README.md` = code→Russian cheat-sheet. Founder created the CVAT Cloud project, imported the schema, and confirmed box-drawing works end-to-end with sample images. Sample field photos in `labeling/samples/` (gitignored).
- **Stage 2 MVP — manual export/import CLIs (built 29 May 2026).** Migration 0006 adds the `labels` table (YOLO-normalized cx/cy/w/h, FK→submissions ON DELETE CASCADE). `labeling/export.py` queries status=ready_for_labeling, packages images + manifest.csv into a zip streamed to stdout, flips status to in_labeling. `labeling/import.py` parses CVAT for Images 1.1 XML from a stdin zip, converts pixel coords → YOLO-normalized, idempotent re-import per submission, flips status to labeled (or leaves at in_labeling if zero boxes). Both run as `docker compose -f docker-compose.prod.yml run --rm -T bot python -m labeling.{export,import}` via SSH. Status flow ready_for_labeling → in_labeling → labeled wired end-to-end. **Stage 2.B (cron) and 2.C (pre-annotation) deferred** — Stage 2.B adds ~30 min when batches become weekly+; Stage 2.C gated on MFWD download + v0 training (see [[newgrain-bootstrap-decisions]]). Usage walkthrough in `labeling/README.md`.
- **Annotation reference sheet + voice (built 10 Jun 2026).** `labeling/reference.py` builds a self-contained HTML (base64 thumbnails) per batch — one card per photo keyed by the CVAT filename `{uuid}.{ext}`, showing: field (or "ВНЕ ПИЛОТА" badge), category, species hint resolved to Latin **+ the CVAT label code** (via new `weed_species.cvat_code`, migration 0008; punctuation-normalized + compound-hint token-split matching), 🎤 RU voice transcript, 🇬🇧 EN translation, 💬 text. `--deliver` uploads it to Object Storage (`reference/…html`) and sends a 7-day presigned link via Telegram text alert — pushing the ~1 MB file through the Cloudflare relay fails (SSL reset on large multipart), a short URL works. Auto-delivered by `pipeline.sh` after each export. **Voice transcription/translation (both Yandex as of 10 Jun 2026):** RU transcript = **Yandex SpeechKit** STT (`bot/transcribe.py`, short-audio sync REST, oggopus/ru-RU; migrated off local faster-whisper — A/B showed SpeechKit better on Russian weed names, e.g. exact "Чина клубненосная" vs Whisper's garbled split; also frees the RAM-tight VM of the 0.5 GB model, lighter image, faster-whisper dep removed, whisper_cache volume dropped; 30 s sync-API cap — long notes would need async later). **EN translation = YandexGPT** (`bot/translate_llm.py`, migration 0009 adds `comment_voice_text_en`), grounded in the weed_species dict so colloquial/ASR-garbled names map to correct Latin ("мышей зелёный"→Setaria viridis); needs `YC_API_KEY`+`YC_FOLDER_ID` in server .env (service account role `ai.languageModels.user`). **Pest/insect category (10 Jun 2026):** new "Вредитель" category + picker (mirrors disease picker; `bot/taxonomy.py` PESTS = 30 grain-pest species, code+RU+Latin+in_picker). Taxonomy from the **Syngenta grain-pest atlas TOC — names only, per LICENSING.md §2.1** (atlas content off-limits; never copy text/photos/recommendations). 15 priority pests → bot picker + CVAT classes now (CVAT → 55 labels: 23 weeds + 11 diseases + 15 pests + 6 stresses); other 15 = dictionary candidate pool via "Другой вредитель", CVAT class on first sighting (data-driven). 4 Latin names tagged "(уточнить)" for Almas. `reference.py` resolves pest hints → Latin+code. **Photo dedup + `duplicate` status (10 Jun 2026):** byte-identical re-uploads by the same agronomist are now caught at upload — `on_field` sha256's the bytes and skips if `submissions.image_hash` (migration 0011, indexed by user_id+hash) already has it. Root incident: during the 8 Jun Telegram-relay outage Almas re-sent photos that seemed to fail → duplicate submissions; 3 dups found+marked `status='duplicate'` (a new free-text status value, excluded from the dataset since it's not `labeled`/`ready_for_labeling`). Dataset = `labels` table joined to submissions; de-duping = mark redundant `duplicate` + delete its redundant label. **Typed comments (comment_text) get the same EN treatment** (migration 0010 `comment_text_en`, inline + backfill, shown as 💬 RU + 🇬🇧 EN in the reference). **First LLM integration** — same key will later power the agronomist-agent. **BUG FIXED 10 Jun:** `db.update_submission` allowed-set was missing `comment_voice_text` → every inline voice transcript was silently dropped (NOT a transient failure as first thought). `bot/backfill_voice.py` (nightly, runs first in pipeline.sh) self-heals missing RU/EN from the saved .ogg. **5 new weed species promoted** (migration 0008, synced to CVAT → 39 labels): euphorbia, taraxacum, artemisia, equisetum, polygonum_aviculare + бодяк alias→cirsium; "Молокан/молочай" pending CAO disambiguation (Lactuca vs Euphorbia).
- **Stage 3 — pre-annotation (deferred; needs ML Lead + GPU).** v0 bootstrap model on public datasets + GroundingDINO/SAM2 fallback; GPU VM (~₽50k/mo). Genuine ML-eng work, can't be done solo/non-technical. Defer until labeling volume justifies it.

**Key blocker right now:** prod DB has **zero submissions** (test photos cleared, agronomist hasn't started). So there's nothing to feed CVAT yet — Stage 2's real value starts once collection produces a batch.

Spec full end-state is heavy (~₽80k/mo: GPU VM, managed PG/Redis, self-host CVAT, DVC) — deliberately doing a lighter pragmatic subset for the Г1 pilot. Builds on [[newgrain-prod-deploy]] and [[newgrain-status-2026-05]].

**Third-party content policy (29 May 2026):** `LICENSING.md` (top of repo, referenced from CLAUDE.md) sets the hard rule that **commercial agrochemical atlases (Syngenta/BASF/Bayer/etc.) are off-limits for any product-facing use** — text, photos, or recommendation tables. The four Syngenta atlases on the founder's machine (weeds, soy diseases, sunflower diseases, cereal pests) are TOC-readable only; recommendations would (a) infringe IP from a direct competitor (Syngenta = Cropwise) and (b) collapse the neutrality moat. Authoritative neutral alternatives for each use case listed in LICENSING.md — most importantly **Государственный каталог пестицидов** as the source of truth for active ingredient recommendations (vs any chem-co catalog). `labeling/class_gaps_from_atlases.md` is the worked TOC analysis: zero soy disease classes in current spec (anchor crop = gap), 2 high-impact sunflower diseases missing (`sunflower_sclerotinia`, `sunflower_downy_mildew`), 2-3 weed candidates for CAO review (`stellaria_media`, `apera_spica_venti`, possibly `alopecurus_myosuroides`). Pests stay out of Г1 (would push past 90-day scope; cereal pest schema flagged for Y2 with Sunn pest + aphids as priority).
